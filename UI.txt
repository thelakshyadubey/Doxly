SECTION 2 — AUTHENTICATION MODEL (read carefully)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

There is NO JWT, no session cookie, no Authorization header.
Auth is Google OAuth 2.0 — but all token storage happens server-side
in Redis. The frontend never sees or handles any token.

The only thing the frontend manages is the user_id:
  - user_id is an opaque string (e.g. the user's email)
  - On first visit, prompt the user to enter their email as user_id
  - Store user_id in localStorage for persistence across sessions
  - Pass user_id in EVERY API request (body, query param, or form field)

The frontend's only auth responsibilities:
  1. Check if the user is already authorized via GET /auth/status
  2. If not, redirect them to GET /auth/login to start the Google OAuth flow
  3. Provide a "Revoke Access" button that calls DELETE /auth/revoke
  4. The /auth/callback route is handled entirely by the backend —
     the frontend must NEVER call it directly

────────────────────────────────────────────────────────────
HEALTH
────────────────────────────────────────────────────────────

GET /health/live
  Purpose  : Liveness probe — is the server process running?
  Request  : none
  Response : { "status": "ok" }
  Usage    : Call on app load. If anything other than "ok" is
             returned, show a full-screen "Server Offline" banner
             and block ALL user interaction until it recovers.

GET /health/ready
  Purpose  : Readiness probe — are all dependencies healthy?
  Request  : none
  Response 200 (all healthy):
    {
      "status": "ok",
      "qdrant": true,
      "neo4j":  true,
      "redis":  true,
      "details": {}
    }
  Response 503 (degraded):
    {
      "status": "degraded",
      "qdrant": false,
      "neo4j":  true,
      "redis":  true,
      "details": { "qdrant": "ping failed" }
    }
  Usage    : Show a persistent status bar on all screens with
             individual green/red dot indicators for Qdrant,
             Neo4j, and Redis. If any is false, show a yellow
             warning banner: "One or more services are degraded."

────────────────────────────────────────────────────────────
AUTH
────────────────────────────────────────────────────────────

GET /auth/status?user_id={user_id}
  Purpose  : Check if user has already authorized Google Drive.
             Call this on every app load before showing any screen.
  Response : { "user_id": "alice", "authorized": true | false }
  Usage    : if authorized=false → show Connect screen
             if authorized=true  → go to Upload screen

GET /auth/login?user_id={user_id}
  Purpose  : Start the Google OAuth 2.0 flow.
             This is a browser redirect, NOT a fetch/axios call.
             Do: window.location.href = `${BASE_URL}/auth/login?user_id=${user_id}`
  Response : 302 redirect to Google's consent screen.
             After user approves, Google redirects to /auth/callback
             automatically — the frontend does NOT handle this.

GET /auth/callback  (DO NOT CALL FROM FRONTEND)
  This is called by Google automatically. After it completes,
  optionally redirect the user to a success page or re-check
  /auth/status to confirm authorization succeeded.

DELETE /auth/revoke?user_id={user_id}
  Purpose  : De-authorize the user, remove stored tokens.
  Response : { "user_id": "alice", "authorized": false, "message": "Tokens revoked." }
  Usage    : Triggered by a "Revoke Access" or "Logout" button.
             After success, clear user_id from localStorage and
             return to the landing/connect screen.

────────────────────────────────────────────────────────────
INGEST
────────────────────────────────────────────────────────────

POST /ingest/upload
  Purpose  : Upload one document image page. OCR runs immediately.
             The full pipeline runs only when /flush is called.
  Request  : multipart/form-data
               user_id  (string, form field, required)
               file     (file, required — JPEG / PNG / TIFF / PDF page)
  Response 202:
    {
      "session_id": "uuid-string",
      "page_count": 1,
      "status": "queued"
    }
  Error responses:
    400 — empty file
    401 — user has not completed OAuth (show re-auth prompt)
    502 — OCR failed (show retry option)
  Usage    : Call once per file. The session_id returned from
             the first upload must be reused for all subsequent
             uploads in the same batch. Always show upload progress
             per file. Accumulate page_count across uploads.

POST /ingest/flush/{session_id}?user_id={user_id}
  Purpose  : Close the upload window and trigger the full AI pipeline:
             classify → coreference resolution → chunking →
             embedding → write to Qdrant + Neo4j.
             This call BLOCKS until the entire pipeline completes.
             It is idempotent — safe to call again if already INDEXED.
  Path     : session_id (string)
  Query    : user_id (string, required)
  Body     : none
  Response 200:
    {
      "session_id": "uuid-string",
      "chunk_count": 24,
      "entity_count": 8,
      "doc_type": "invoice",
      "status": "indexed"
    }
  Error responses:
    401 — user has not completed OAuth
    404 — session not found or expired
    422 — pipeline validation error
    500 — pipeline failed
  Usage    : While waiting, show a full animated processing state
             (spinner or step progress: classify → chunk → embed → store).
             On success, show doc_type detected + chunk_count + entity_count
             in a success card, then offer to proceed to the Query screen.

────────────────────────────────────────────────────────────
QUERY
────────────────────────────────────────────────────────────

POST /query
  Purpose  : Ask a question. Returns a complete answer with citations.
             Uses hybrid vector + graph retrieval, then Gemini generates
             the answer. Waits for full response before returning.
  Request body (JSON):
    {
      "user_id": "alice",
      "query":   "What is the total amount on the invoice?",
      "filters": {
        "doc_type":   "invoice",       (optional — DocType enum)
        "session_id": "uuid-string"    (optional — restrict to one session)
      }
    }
  Response 200:
    {
      "answer":     "The total amount is $1,200.",
      "citations":  [
        {
          "source":   "My Drive/DocumentIntelligence/session-id/invoice/page_1.jpg",
          "page":     1,
          "chunk_id": "uuid-string"
        }
      ],
      "confidence": 0.87
    }
  Error: 500 — retrieval or generation failed

GET /query/stream
  Purpose  : Same as POST /query but streams the answer token-by-token
             using Server-Sent Events (SSE).
  IMPORTANT: filters must be passed as a JSON-encoded string, not an object.
  Query params:
    user_id  (string, required)
    query    (string, required)
    filters  (string, optional) — JSON-encoded filters object
               Example: filters={"doc_type":"invoice","session_id":"uuid"}
  Implementation:
    const params = new URLSearchParams({ user_id, query });
    if (filters) params.append('filters', JSON.stringify(filters));
    const es = new EventSource(`${BASE_URL}/query/stream?${params}`);
    es.onmessage = (e) => {
      if (e.data === '[DONE]') { es.close(); return; }
      appendTokenToAnswer(e.data);
    };
  Response : text/event-stream
    Each event : data: {token text}
    Final event: data: [DONE]
  Error: 422 — invalid filters JSON | 500 — retrieval failed

SECTION 4 — ALL DATA MODELS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DocType (string enum — use exactly these values in dropdowns):
  "invoice" | "contract" | "letter" | "form" |
  "note"    | "report"   | "receipt"| "other"

SessionStatus (string enum — used for badge display):
  "open"       → blue    (session accepting uploads)
  "queued"     → yellow  (upload window closed, pipeline not started)
  "processing" → orange  (pipeline is running)
  "indexed"    → green   (fully stored in Qdrant + Neo4j — success)
  "failed"     → red     (pipeline error)

UploadResponse:
  session_id  : string
  page_count  : integer
  status      : SessionStatus  (always "queued" on upload)

FlushResponse:
  session_id  : string
  chunk_count : integer
  entity_count: integer
  doc_type    : DocType
  status      : SessionStatus  (always "indexed" on success)

QueryRequest:
  user_id     : string   (required)
  query       : string   (required, min length 1)
  filters     : QueryFilters (optional)

QueryFilters:
  doc_type    : DocType | null  (optional)
  session_id  : string | null   (optional)

QueryResponse:
  answer      : string
  citations   : Citation[]
  confidence  : float  [0.0 – 1.0]

Citation:
  source      : string   (Google Drive path of the source document)
  page        : integer
  chunk_id    : string   (UUID)

ErrorResponse (returned on ALL 4xx and 5xx responses):
  error       : string   (machine-readable code, e.g. "internal_server_error")
  message     : string   (human-readable — ALWAYS display this in the UI)
  detail      : any | null

SECTION 6 — ERROR HANDLING (apply everywhere)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

All errors from the API follow this shape:
  { "error": "machine_code", "message": "Human readable text", "detail": any | null }

Rules:
  - ALWAYS display the `message` field to the user (toast or inline)
  - Log `error` and `detail` to the browser console for debugging
  - Never show raw status codes to the user

Specific cases:
  400 (bad file)      → inline error below upload zone
  401 (not authorized)→ show re-auth prompt with "Connect Google Drive" button
  404 (session gone)  → "Session not found. Please start a new upload."
  422 (validation)    → show message inline near the triggering input
  500 (server error)  → toast: show message, offer retry button
  502 (OCR failed)    → "OCR failed for this page. Try re-uploading." + retry
  503 (degraded)      → already handled by status bar
